This sample illustrates using Node.js as a proxy between the client application and the Logentries servers.The client sends the log message via AJAX to a pre-defined route on the server. That server based route send the message onto Logentries. In order to maximize portability and security, the token is kept in a seperate json file that could be encrypted or secured as long as the user account running the node server has access.
To see the sample in action:
  1. Update the token.json file by replacing the placeholder text with the target logs token.
  2. Start the server by running "node server.js". The server will be accessible on port 3000. Note: make sure to run npm install in the project directory before starting the server.
  3. Open a web browser to http://localhost:3000.
  4. Choose a logging severity, then enter a message in the text field.
  5. Press the "Send Log" button.
  6. Check your log in a few minutes to see the new log entry.
